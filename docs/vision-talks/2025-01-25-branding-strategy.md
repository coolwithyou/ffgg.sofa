## 1. 브랜드 아이덴티티

### 서비스명

**faq.guide**

- 도메인이 곧 브랜드명
- 별도의 서비스명 없이 URL 자체가 이름
- 발음: "에프에이큐 가이드" 또는 "팩 가이드"

### 태그라인 후보

| 태그라인 | 느낌 |
|----------|------|
| **"고객 질문, AI가 답합니다"** | 직접적, 기능 중심 |
| **"쉽게 만드는 FAQ"** | 쉬움 강조 |
| **"당신의 FAQ 가이드"** | 소유감 |
| **"질문은 고객이, 답변은 AI가"** | 역할 분담 명확 |
| **"5분 만에 만드는 AI FAQ"** | 속도 강조 |

### 핵심 가치 (3가지)

1. **쉬움** - 누구나 5분 안에 시작
2. **똑똑함** - AI가 맥락을 이해하고 답변
3. **친근함** - 챗봇 같지 않은 자연스러운 대화

## 2. 포지셔닝

### 우리는 무엇인가

> **"복잡한 제품을 파는 소규모 기업을 위한 AI FAQ 가이드"**

### 우리는 무엇이 아닌가

| 우리가 아닌 것 | 이유 |
|----------------|------|
| CS 챗봇 (채널톡) | 컴플레인 처리/상담이 아님 |
| 지식 관리 도구 (노션) | 내부용이 아닌 고객용 |
| 홈페이지 빌더 | 웹사이트 전체가 아닌 FAQ 특화 |
| RAG 플랫폼 (기술 회사용) | 비개발자를 위한 서비스 |

### 경쟁 구도

```
                    전문적/복잡
                        ↑
                        │
       Chatbase ●       │       ● 채널톡 ALF
                        │
    ────────────────────┼────────────────────→ 비싸다
                        │
                        │  ● faq.guide
                        │    (여기!)
                        │
                        ↓
                   쉬움/친근
```

## 3. 타겟 고객

### 1차 타겟: 옻칠 기업 (파일럿)

- 복잡한 제품 지식
- 초보 고객이 많음
- 1:1 설명에 시간 소모

### 확장 타겟

| 영역 | 예시 | 공통점 |
|------|------|--------|
| 취미/공예 | 가죽공예, 도자기, 레진아트 | 진입장벽 높음 |
| 전문 장비 | 커피 머신, 오디오, 3D 프린터 | 사용법 복잡 |
| 건강/웰니스 | 필라테스 기구, 보충제 | 개인화 설명 필요 |
| 행사/이벤트 | 아트페어, 전시회, 컨퍼런스 | 다양한 정보 안내 |

### 고객 페르소나

**"바쁜 소규모 기업 담당자"**
- IT 전문가가 아님
- CS 전담 인력 없음
- 같은 질문에 반복 답변하느라 지침
- "챗봇? 어렵고 비싸지 않아?"라고 생각

## 4. 브랜드 톤 & 보이스

### 톤

| O (이렇게) | X (이렇게 아님) |
|------------|-----------------|
| 친근하고 가벼움 | 딱딱하고 기업적 |
| 쉬운 단어 | 전문 용어 |
| 짧은 문장 | 긴 설명 |
| 능동적 ("해보세요") | 수동적 ("됩니다") |

### 예시

```
❌ "데이터셋을 생성하고 문서를 업로드하여 청크를 임베딩합니다"
✅ "파일 올리면 끝. AI가 알아서 정리해요"

❌ "RAG 기반 하이브리드 검색으로 정확한 답변을 제공합니다"
✅ "고객이 뭘 물어봐도 똑똑하게 답해요"

❌ "콘솔에서 챗봇 설정을 구성하세요"
✅ "스튜디오에서 내 페이지를 꾸며보세요"
```

### 피해야 할 단어

| 피할 단어 | 대체 단어 |
|-----------|-----------|
| 데이터셋 | (사용 안 함) |
| 청크 | (사용 안 함) |
| 임베딩 | (사용 안 함) |
| RAG | (사용 안 함) |
| 콘솔 | 스튜디오 |
| 챗봇 | AI, 가이드 |
| 발행 | 공개하기 |
| 테넌트 | (사용 안 함) |

## 5. 비주얼 아이덴티티

### 로고 방향

**faq.guide** 텍스트 로고 (심플)

```
faq.guide
```

- 소문자 유지
- Pretendard Bold
- "." 을 포인트 컬러로 강조 가능

### 컬러 팔레트 (확정)

| 용도 | 컬러 | HEX | 의미 |
|------|------|-----|------|
| Brand | Coral | `#FF6B4A` | 친근함, 에너지, CTA |
| Primary | Black/White | `#1A1A1A` | 기본 텍스트, 버튼 |
| Success | Green | `#22C55E` | 성공, 완료 |
| Warning | Yellow | `#F59E0B` | 주의, 안내 |
| Background | Off-white | `#FCFCFC` | 깔끔함, 여백 |

→ 상세 정의: [2025-01-25-design-tokens.md](./2025-01-25-design-tokens.md)

### 타이포그래피

**Pretendard Variable 단일 사용**

- 장시간 가독성을 위해 단일 폰트 통일
- 문서를 다루는 서비스 특성 반영
- 웨이트와 크기로 위계 표현

### 디자인 원칙

1. **여백 충분히** - 답답하지 않게
2. **큰 글씨** - 읽기 쉽게
3. **둥근 모서리** - 부드러운 인상
4. **최소한의 요소** - 선택지 3개 이하

### 상세 문서

- UI/UX 철학: [2025-01-25-ui-ux-philosophy.md](./2025-01-25-ui-ux-philosophy.md)
- 디자인 토큰: [2025-01-25-design-tokens.md](./2025-01-25-design-tokens.md)

## 7. URL 구조

### 확정 구조

```
faq.guide                    → 랜딩 페이지
faq.guide/[회사명]           → 공개 FAQ 페이지
faq.guide/studio             → 관리 화면 (로그인)
faq.guide/studio/guidebook   → 가이드북 관리
faq.guide/studio/page        → 내 페이지 꾸미기
faq.guide/studio/settings    → 설정
```

### 예시

```
faq.guide/ottchil            → 옻칠 기업의 공개 페이지
faq.guide/coffeelab          → 커피랩의 공개 페이지
faq.guide/artfair2025        → 아트페어의 공개 페이지
```
