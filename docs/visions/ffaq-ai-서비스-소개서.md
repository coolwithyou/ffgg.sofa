## 1. Executive Summary

### 한 줄 정의

**"행사 문서를 넣고 QR을 붙이면, 관람객이 현장에서 바로 답을 얻는 AI 안내원(Concierge) 챗봇"**

### 핵심 메시지

ffaq.ai는 **마라톤, 팝업스토어, 아트페어, 컨퍼런스** 등 단기 행사 운영자를 위한 **QR 기반 대화형 안내 자동화 서비스**입니다.

기존 FAQ 챗봇 솔루션이 "상시 CS 운영"에 최적화되어 있다면, ffaq.ai는 **"행사 기간에만 열고, 끝나면 자동으로 닫히는"** 스케줄형 운영에 특화되어 있습니다.

### 핵심 가치 제안

| 가치 | 설명 |
|------|------|
| **즉시 배포** | 문서 업로드 후 10분 내 행사 전용 챗봇 오픈 |
| **QR 씬(Scene)** | 주차/환불/동선 등 맥락별 QR로 정확한 답변 제공 |
| **스케줄 운영** | 행사 기간에 맞춰 자동 오픈·만료 |
| **브랜드 경험** | 행사 톤앤매너에 맞는 UI 커스터마이징 |
| **독립 랜딩** | 웹사이트 없이도 행사 안내 페이지 즉시 생성 |

### 성능 지표

| 지표 | 수치 |
|------|------|
| 평균 응답 시간 | **2.5초** (업계 평균 5-10초) |
| 검색 정확도 | **Faithfulness 0.9+** |
| 한국어 성능 | 영어 대비 **+13.4%** 향상 |

## 2. 문제 정의와 시장 기회

### 2.1 행사 운영의 구조적 문제

단기 행사에서 반복되는 현실:

```
😤 안내문/주의사항/FAQ가 이미 있어도...
   ↓
📱 참가자는 끝까지 읽지 않음
   ↓
📞 "내 상황에도 해당하나요?" 전화/DM 폭주
   ↓
🔥 행사 직전/당일 문의 폭증
   ↓
😵 운영팀은 동선/안전/현장 대응에 집중 불가
```

**핵심 인사이트**: 사람들은 "찾기"보다 **"확신"**을 원합니다.
내 상황에 해당하는지 빠르게 확인하려고 직접 문의합니다.

### 2.2 기존 솔루션의 미스매치

| 기존 솔루션 | 문제점 |
|------------|--------|
| **채널톡/해피톡 등** | 상시 CS 운영 중심, 단기 행사에 온보딩이 무거움 |
| **시나리오 챗봇** | 구축비 500만원+, 시나리오 수정마다 개발자 필요 |
| **콜센터/상담사** | 인건비 월 200만원+, 수요 증가 시 확장 어려움 |
| **단순 FAQ 페이지** | 검색 불편, 자연어 미지원, 고객의 30%만 활용 |

**단기 행사가 원하는 것:**
- ✅ 빠른 오픈/회수
- ✅ 행사 단위 과금 (예측 가능)
- ✅ QR 씬 분기와 현장 UX
- ✅ '행사용' 템플릿과 운영 모드

## 3. 솔루션 개요

### 3.1 제품 비전

> **"안 읽고 전화하는 문의를 현장에서 즉시 해결하여,
> 운영 리스크와 혼선 비용을 낮춘다."**

### 3.2 작동 원리

```
┌─────────────────────────────────────────────────────────────┐
│                        ffaq.ai 플로우                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   📄 문서 업로드          🤖 AI 자동 학습         🚀 즉시 배포   │
│   (안내/정책/FAQ)    →    (RAG 인덱싱)      →    (QR 생성)    │
│                                                             │
│                           ↓                                 │
│                                                             │
│   📱 현장 QR 스캔    →    💬 맥락 인식 답변   →   ✅ 확신 제공   │
│   (주차? 환불?)          (Intent-Aware)         (즉시 해결)   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.3 핵심 가치 (스테이크홀더별)

#### 운영자 관점
- 반복 문의 대응 시간을 줄여 **현장 운영 품질** 상승
- 공지 전달 실패/혼선을 줄여 **클레임·리스크** 감소
- 행사 규모가 커질수록 효과 증대 (문의 스파이크에 강함)

#### 참가자/관람객 관점
- **"내 상황"**에 맞는 답을 즉시 수신 (확신 제공)
- 현장에서 줄 서서 묻지 않아도 됨 (접근성)

#### 브랜드 관점
- 세련된 인터페이스와 즉시 응답으로 **브랜드 경험** 개선
- "안내가 잘 되어 있는 행사"라는 인상 형성

## 4. 핵심 기술

ffaq.ai는 검증된 RAG(Retrieval-Augmented Generation) 기술 위에 행사 특화 기능을 구축했습니다.

### 4.1 Hybrid Retrieval (Dense + Sparse)

단일 검색 방식의 한계를 극복하는 융합 검색:

| 검색 방식 | 기술 | 장점 | 가중치 |
|----------|------|------|--------|
| **Dense Search** | pgvector + BGE-m3-ko | 의미적 유사도 | 0.7 |
| **Sparse Search** | PostgreSQL FTS + Nori | 키워드 정확 매칭 | 0.3 |

**RRF (Reciprocal Rank Fusion)** 알고리즘으로 두 결과를 최적 융합:

```
RRF Score = Σ 1 / (k + rank_i)
```

### 4.2 Intent-Aware RAG

질문 의도 분류 기반 응답 전략으로 할루시네이션 최소화:

```
사용자 질문
    ↓
┌─────────────────────────────────┐
│       Intent Classifier         │
├─────────────────────────────────┤
│ CHITCHAT     → 간단한 인사 응답   │
│ DOMAIN_QUERY → RAG 검색 + 생성   │
│ OUT_OF_SCOPE → 범위 외 안내      │
└─────────────────────────────────┘
    ↓
Query Router (Intent + RAG 결과 조합)
    ↓
최적 응답 생성
```

### 4.3 한국어 최적화

| 기술 | 효과 |
|------|------|
| **BGE-m3-ko 임베딩** | 영어 대비 +13.4% 성능 향상 |
| **Nori 형태소 분석** | 정확한 한국어 키워드 검색 |
| **한국 서비스 연동** | 카카오톡, 네이버, PortOne 지원 |

### 4.4 응답 시간 최적화

| 단계 | 최적화 결과 |
|------|------------|
| 캐시 조회 | 80ms (800ms 절감) |
| Query Rewriting | 첫 질문 스킵 (1,115ms 절감) |
| HNSW 벡터 인덱스 | 50ms (10만 청크 기준) |
| **총 응답 시간** | **~2.5초** |

## 5. 제품 구성

ffaq.ai는 3가지 핵심 제품으로 구성됩니다:

### 5.1 Event Bot (행사 전용 챗봇)

문서 기반 자동 학습 + FAQ Builder

```
┌─────────────────────────────────────────┐
│              Event Bot                   │
├─────────────────────────────────────────┤
│                                         │
│  📥 입력                                 │
│  └── PDF/문서/공지/FAQ 업로드            │
│                                         │
│  ⚙️ 처리                                 │
│  ├── 스마트 청킹 (의미 단위 분리)         │
│  ├── 임베딩 생성 (BGE-m3-ko)            │
│  └── 품질 점수 자동 평가                 │
│                                         │
│  🎛️ 설정                                 │
│  ├── FAQ Builder (자주 묻는 질문 보강)   │
│  ├── 금지/주의 답변 규칙                 │
│  └── 페르소나 (말투, 역할, 제약조건)      │
│                                         │
└─────────────────────────────────────────┘
```

**지원 문서 포맷:**
- Markdown (.md)
- 텍스트 (.txt)
- PDF (.pdf)
- Word (.docx)

### 5.2 QR Scenes (맥락별 진입점)

행사장 곳곳에 QR을 배치, 각 QR마다 다른 **씬(Scene)**으로 진입:

```
┌─────────────────────────────────────────────────────────────┐
│                     QR Scenes 예시                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  🅿️ 주차/교통        🎫 입장/티켓        💸 환불/교환        │
│  ┌─────────┐        ┌─────────┐        ┌─────────┐        │
│  │ [QR 1]  │        │ [QR 2]  │        │ [QR 3]  │        │
│  └─────────┘        └─────────┘        └─────────┘        │
│                                                             │
│  📦 분실물           📋 프로그램         🗺️ 부스 위치        │
│  ┌─────────┐        ┌─────────┐        ┌─────────┐        │
│  │ [QR 4]  │        │ [QR 5]  │        │ [QR 6]  │        │
│  └─────────┘        └─────────┘        └─────────┘        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**기대 효과:**
- 참가자가 긴 안내문에서 찾지 않아도 됨
- 챗봇이 이미 문맥을 알고 답변 → 정확도/속도 상승

### 5.3 Event Landing (독립 랜딩 페이지)

웹사이트 없이도 바로 공유 가능한 행사 페이지:

```
┌─────────────────────────────────────────┐
│           Event Landing Page            │
├─────────────────────────────────────────┤
│                                         │
│  📍 상단: 행사 핵심 정보                  │
│  ├── 시간/장소                          │
│  ├── 티켓 정보                          │
│  └── 주요 공지사항                       │
│                                         │
│  💬 중단: AI 챗봇                        │
│  └── 바로 질문하기                       │
│                                         │
│  📞 하단: 운영자 연결                    │
│  ├── 문의 폼                            │
│  └── 약관/주의사항                       │
│                                         │
└─────────────────────────────────────────┘
```

**커스터마이징 옵션:**
- 브랜드 색상/폰트/로고
- 커스텀 도메인/서브도메인
- 다크/라이트 모드 자동 대응

### 5.4 스케줄 기반 운영 (단기 행사 특화)

행사 기간/시간을 지정하여 자동 운영:

```
시간 →

D-14        D-Day         D+3
  │           │            │
  ▼           ▼            ▼
┌───────────────────────────────────────┐
│ 오픈       피크 모드      종료/만료    │
│                                       │
│ • 챗봇    • 캐시 강화    • 챗봇 비노출  │
│   활성화  • 간소화 응답  • 종료 안내    │
│          • 트래픽 대응  • 아카이브     │
└───────────────────────────────────────┘
```

**만료 시 동작 옵션:**
- 챗봇 비노출 또는 "행사 종료 안내"로 자동 전환
- 링크/QR 유지 여부 선택 (아카이브, 재사용)

### 5.5 운영 대시보드

행사 운영에 바로 쓰이는 실시간 지표:

| 지표 | 용도 |
|------|------|
| 시간대별 문의량 추이 | 피크 시간대 파악 |
| TOP 질문 10 | 자주 묻는 질문 확인 |
| 답변 불가/불확실 목록 | 문서 보강 포인트 발견 |
| 오답 신고/불만 키워드 | 품질 개선 대응 |

**자동 제안 기능:**
- "이 질문은 문서 보강 필요" 가이드 자동 생성
- 문서/FAQ에 추가할 문장 AI 제안

### 5.6 리스크 방지 장치

행사에서 필수인 안전 장치:

| 장치 | 설명 |
|------|------|
| **출처/근거 표시** | 문서 인용/링크 옵션 |
| **확신도 표시** | 낮을 때 "확실하지 않음" + 스태프 연결 유도 |
| **금지 답변 정책** | 안전/법/의료 등 위험 카테고리 고정 문구 |

## 6. 사용자 여정

### 6.1 운영자 여정 (Setup → Run → Archive)

```
1. 행사 생성
   └── 이름/기간/타임존 설정

2. 문서 업로드
   └── 안내/정책/FAQ 파일

3. 자동 학습 완료
   └── "테스트 질문"으로 검증

4. 씬 생성
   └── 주차/환불/동선 등 → QR 자동 생성

5. 랜딩 페이지 발행
   └── 커스텀 도메인/서브도메인 선택

6. 스케줄 설정
   └── 오픈/피크/만료 시점 지정

7. 행사 중 모니터링
   └── TOP 질문/미응답 확인 → 실시간 문서 보강

8. 행사 종료 후
   └── 아카이브/복제 (다음 행사 재사용)
```
